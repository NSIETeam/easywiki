核心功能

从知识录入到智能检索，全链路覆盖团队协作场景

📝

决策与经验记录

支持页面树结构，Agent 自动提取会话中的决策、Bug修复、最佳实践，经人工审核后入库。

🔍

全文搜索 (FTS5)

基于 SQLite FTS5 的中文全文检索，秒级定位文档内容，支持关键词高亮和语义相似度排序。

🕸️

知识图谱

自动抽取文档中的实体关系，构建可视化力导向图，直观展示项目、决策、人员之间的关联。

🤖

Agent 活动中心

AI Agent 自动记录工作过程，待审核条目统一管理，支持批量批准、驳回、编辑后入库。

🔒

并发控制与版本历史

乐观锁防止多人编辑覆盖，每次修改自动生成版本快照，支持一键回滚到任意历史版本。

🛡️

安全与合规

HTML 输入消毒防 XSS，PII 敏感信息检测，操作审计日志，API 限流 60 次/分钟。

💾

自动备份

每日自动 SQLite 数据库备份，保留最近 7 天，WAL checkpoint 确保数据一致性。

📊

进度同步表

可自定义字段的项目进度看板，实时同步关键字段，支持文本、日期、枚举、百分比类型。

🔔

Webhook 通知

文档变更、Agent 录入、冲突检测等事件支持 Webhook 推送，可对接飞书、钉钉等 IM。

快速开始

三种方式，30 秒内启动

Docker
# 一键启动（推荐）
docker run -d \
  -p 8080:8080 \
  -v easywiki-data:/data \
  -e ORGMIND_ADMIN_PASSWORD=admin123 \
  easywiki/easywiki

# 访问 http://localhost:8080
# 登录 admin@local / admin123
Python
# 从源码运行
git clone https://github.com/NSIETeam/easywiki.git
cd easywiki
pip install -r requirements.txt
python -m uvicorn orgmind.main_sqlite:app --host 0.0.0.0 --port 8080

常见问题

EasyWiki 需要什么数据库？

零外部依赖。使用 SQLite 作为存储引擎，内置向量相似度计算和 FTS5 全文搜索，无需 PostgreSQL 或 Redis。

数据存储在哪里？

所有数据存储在本地 SQLite 文件中（默认 ~/.easywiki/easywiki.db）。Docker 部署时挂载 /data 卷即可持久化。每日自动备份保留 7 天。

支持多少人同时使用？

SQLite WAL 模式 + 乐观锁支持中小团队（50人以内）协作。每个文档保留 20 个版本快照，支持冲突检测和回滚。

AI Agent 如何工作？

Agent 在开发过程中自动提取决策、最佳实践、Bug 修复等知识条目，写入待审核队列。人工审核通过后正式入库，确保质量。

支持哪些部署方式？

Docker 一键部署、Python 源码运行、Docker Compose 编排。支持非 root 用户运行，内置健康检查端点。

是否开源？

完全开源，代码托管在 GitHub：github.com/NSIETeam/easywiki。欢迎 Issue 和 PR。
